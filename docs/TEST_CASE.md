# 테스트 케이스 명세서 (TEST_CASE) — Disaster Alert Archive

> [REQUIREMENTS.md](./REQUIREMENTS.md)의 FR/NFR 단위 요구사항에 대응하는 테스트 케이스다. **현재 자동화 커버리지가 매우 낮으므로**(1절 참고), 대부분은 자동화 여부를 "없음"으로 정직하게 표기하고 수동 확인 절차를 기술한다.

## 1. 현재 자동화된 테스트 현황

### 1.1 백엔드 (`backend/src/test`, JUnit5 + `@SpringBootTest`, 실제 테스트 DB 필요)
| 파일 | 테스트 메서드 | 대상 |
|---|---|---|
| `BackendApplicationTests` | `contextLoads` | 스프링 컨텍스트 정상 기동 |
| `DisasterDataInitializerTest` | `testDisasterDataInitialization` | 초기 재난 데이터 적재 |
| `DisasterOpenApiClientTest` | `fetchData` | 행안부 공공 API 클라이언트 호출 |
| `AuthServiceTest` | `reissue`, `이메일_인증후_회원가입_성공`, `이메일_인증안된_상태에서_회원가입_실패` | 토큰 재발급, 이메일 인증 기반 회원가입 |
| `DisasterAlertServiceTest` | `saveData`, `searchAlerts_*`(조건없이/지역명/기간/키워드/종합), `getStats_*`(조건없이/지역/기간/키워드/종합) | 재난 문자 저장 및 검색·통계 조회 |

**커버리지 범위**: 인증(일부), 재난 문자 검색/통계, 초기 데이터 적재. **이벤트 클러스터링, 위험도 계산, FCM 발송, 커뮤니티/댓글, 관심지역, 다국어 번역, 공개 API, 관리자 기능은 자동화 테스트 없음.**

### 1.2 프론트엔드
`frontend/package.json`에 테스트 러너(Jest/Vitest/Playwright 등)가 구성되어 있지 않고, 테스트 파일도 존재하지 않는다. **프론트엔드 자동화 테스트는 전무한 상태.**

## 3. 커버리지 갭 요약

- 백엔드: 인증 일부, 재난 문자 검색/통계만 자동화됨. **이벤트 클러스터링, 위험도, FCM, 커뮤니티, 공개 API, 관리자 기능은 자동화 테스트 0건.**
- 프론트엔드: 자동화 테스트 프레임워크 자체가 없음.
- 가장 시급한 자동화 후보: TC-SANC-03(관리자 API 인증 누락, 실제 보안 결함), TC-NOTI-05(과거 회귀 이력 있는 중복 알림 버그).
